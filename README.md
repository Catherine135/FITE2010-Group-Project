# Elevator Waiting-Time Options

A full-stack smart contract project for trading CALL/PUT options on elevator waiting time.

This repository includes:
- Solidity contracts (token, option instance, factory)
- Hardhat test suite (happy path + failure path + event coverage)
- A lightweight web console for end-to-end demo
- Local deployment tooling for reproducible demos

## Overview
This project models a real-world event (elevator waiting time) as an on-chain derivative.

Actors and settlement logic:
- Writer locks PU token collateral.
- Buyer pays PU premium to buy the option.
- Oracle/Admin records actual waiting time after expiry.
- Settlement:
  - CALL: exercisable when `actualWaitTime > strikeTime`
  - PUT: exercisable when `actualWaitTime < strikeTime`

## Architecture
The system has three core contracts and one demo frontend:

1. `PrintToken` (`PU`)
- ERC20-like settlement token
- owner/minter/quota controls

2. `ElevatorOptionFactory`
- Creates option instances
- Enforces creation rules (duration/premium/collateral/strike/type)
- Handles admin actions (rules, fee withdrawal, relay operations)

3. `ElevatorOption`
- Single option instance state machine
- Handles deposit, buy, oracle record, exercise, cancellation, and retrieval

4. `web/`
- Browser-based console for wallet connect, create/read option, and operational actions

## Contract Design

### 1) PrintToken (`contracts/PrintToken.sol`)
Responsibilities:
- Token accounting (`balanceOf`, `transfer`, `approve`, `transferFrom`)
- Controlled minting:
  - owner unlimited mint
  - optional minter + quota
- One-time airdrop support

Key functions:
- `mint`
- `setMinter`
- `oneTimeAirdrop`
- `transferOwnership` / `acceptOwnership`

### 2) ElevatorOptionFactory (`contracts/ElevatorOptionFactory.sol`)
Responsibilities:
- Create/index option contracts
- Validate creation parameters
- Maintain writer option lists + pagination
- Admin controls (rules, fees, relay actions)

Key functions:
- `createOption`
- `setRules`
- `relayRecordActualTime`
- `relayChangeOracle`
- `getAllOptionsBatch` / `getWriterOptionsBatch`
- `withdrawFees`
- `transferAdmin` / `acceptAdmin`

### 3) ElevatorOption (`contracts/ElevatorOption.sol`)
Responsibilities:
- Single-option lifecycle transitions
- Role-based checks (writer, buyer, oracle/relay)
- Option settlement

Key functions:
- `deposit`
- `buyOption`
- `cancelIfUnbought`
- `cancelIfInactive`
- `changeOracle`
- `recordActualTime`
- `exercise`
- `retrieveExpired`

Recent correctness hardening includes:
- `changeOracle` authority aligned with oracle/relay path
- `newOracle` cannot be writer or buyer
- `premium < collateral` enforced at creation
- Boundary fixes around cancellation/retrieval timing paths

## Option Lifecycle
Recommended operational flow:
1. Deploy `PrintToken`
2. Deploy `ElevatorOptionFactory`
3. Mint/distribute PU to writer and buyer
4. Writer creates option via factory
5. Writer approves and deposits collateral
6. Buyer approves and buys option
7. Wait until expiry
8. Oracle/Admin records actual waiting time
9. Buyer exercises if condition met
10. Otherwise writer retrieves expired collateral:
  - Active option: retrieval requires oracle/admin time to be recorded first
  - Unbought option: writer can retrieve after expiry

## Local Development

### Prerequisites
- Node.js LTS recommended (project works in current setup; Hardhat may warn on non-LTS versions)

### Install
```bash
npm install
```

### Compile
```bash
npm run compile
```

### Run tests
```bash
npm run test
```

### Focused regression subset
```bash
npm run test:focus
```

## Frontend Demo

### 1) Start local chain
```bash
npm run node
```

### 2) Deploy locally and generate frontend config
```bash
npm run deploy:local
```
This generates `web/deployments.localhost.json` with:
- `elevatorOptionFactory`
- `printToken`
- demo account metadata (`demoWriter`, `demoBuyer`)

### 3) Start web server
```bash
npm run serve:web
```
Open:
- `http://127.0.0.1:8000/web/index.html`

### 4) In UI
1. Click **Load Local Deployment**
2. Connect MetaMask (network `31337` / Hardhat local)
3. Run create/deposit/buy/record/exercise workflow
4. Oracle change in UI is relayed through factory admin (`relayChangeOracle`)

## Security Notes
Implemented controls include:
- Reentrancy guard on critical state-changing token-transfer paths
- Role checks for writer/buyer/oracle/relay/admin
- Parameter validation for create and lifecycle transitions
- CEI pattern in cancellation/settlement flows
- Explicit revert reasons for invalid state transitions
